# 現場タスク管理アプリ — CLAUDE.md

Claude Codeで開発を継続するための引き継ぎドキュメント。

---

## プロジェクト概要

建設現場からの仕事依頼を管理するPWA（Progressive Web App）。
スマートフォンのホーム画面に追加して使うことを想定している。

**主な機能：**
- タスク（仕事依頼）の登録・編集・削除
- ステータス管理（未着手 / 進行中 / 完了）
- 入力項目は **現場名 / 人数 / 日付 / 作業内容・メモ** の4つ（件名は廃止）
- 日付指定（期間指定 or 候補日）
- 音声入力（全項目に個別の🎤ボタン。現場名・人数・作業内容はテキスト追記、日付は話した内容を解析して反映）
- **音声一括登録**：話した内容をブラウザ内のローカル解析（ルールベース）で構造化し、現場名・人数・日付・作業内容を自動でフォームに反映（APIキー不要・無料）
- データはlocalStorageに永続化
- オフライン対応（Service Worker）

### フェーズ6：スマホでの音声重複を修正（2026年6月5日）

**問題：** PC版Chromeでは正常だが、スマホ（Android/iOSのChrome・Safari）の一括登録で、一度しか話していない語が大量に重複する（例：「サーパス柳ヶ崎の現場」が7回繰り返し）。フェーズ5の「0番から組み立て直す」修正でもスマホでは直らなかった。

**原因：** モバイルのWeb Speech APIは `continuous=true` を正しく実装しておらず、**確定済みの語を毎回再報告**する。そのため `e.results` 配列自体が同じ確定結果で膨らみ、0番から組み立て直しても重複が残る。`continuous=true` がそもそもモバイル非互換。

**対策（`startAiRec`）：** 一括登録を `continuous=true` から **`continuous=false`（1発話ごとに確定）＋ `onend` で自動再開** 方式に変更。
- 1発話ごとに認識セッションが完結するので配列に重複が溜まらない。
- ユーザーが停止するまで `onend` で次のセッションを `start()` し直し、実質的な連続認識を実現（長文対応）。
- 停止判定は `aiStopRef`（ref）、セッションをまたぐ確定テキストは `aiAccRef`（ref）に蓄積。
- `no-speech` / `aborted` エラーは無視して再開に任せる。
- `stopAiRec` は `aiStopRef.current=true` にしてから `stop()`（→ `onend` で自動再開せず解析へ）。

**教訓：** モバイル対応では `continuous=true` を使わない。**`continuous=false` ＋ 自動再開** が定番の回避策。

**反映時の注意（重要）：** PWAのService Workerが `index.html` をキャッシュするため、スマホでは普通に開くと古いまま。アップロード後はキャッシュ削除が必要（iOS：アプリ削除→再追加 or Safari履歴消去 / Android：Chromeキャッシュ削除 or PWA入れ直し）。

### フェーズ7：現場名判定の改善・人数表現の拡張・カードのフォント調整（2026年6月5日）

**前提（ユーザー情報）：** 現場名はほぼ**マンション名**。発話は基本的に**現場名を最初**に言う。

**変更要件：**
1. 「○○の現場」と言わなくても現場名として判定したい（最初に話す語が現場名）。
2. タスクカードの現場名と作業内容のフォントサイズを入れ替えたい。

**変更内容（`parseVoiceLocal`）：**
- **現場名の「最初の語」フォールバック：** キーワードにも「○○の現場」にも当たらない場合、抽出済み（人数・日付）を除いた**残りの最初のセグメントを現場名**にする（`if(!out.siteName && segs.length){ out.siteName = segs.shift(); }`）。サーパス／プロムナーデ／ファミール／カルム等の接尾語なしブランド名マンションはこれで拾う。
- **現場名キーワードの整理：** `塗装/電気/設備/建材` は作業内容にも頻出する語のため**除外**（「外壁塗装」が現場名と誤判定されていた）。
- **マンション種別語を追加：** `マンション/ハイツ/コーポ/レジデンス/メゾン/ハイム/パレス/ヒルズ`（作業内容にまず出てこない語のみ厳選。`タワー/ガーデン/テラス`等は作業語と被りうるため不採用）。正規表現は `/[^\s、。]*?(?:…種別語…)[^\s、。]*/` とし、接尾語の前後（「パークハイツ城東」の「城東」等）も取り込む。これにより文の途中で言っても検出可。
- **人数表現に「から」を追加：** `\d+\s*(?:[〜～~ー－\-]|から)\s*\d+\s*人…` とし「3から4人」を1語として認識。表示は `から`→`〜` に正規化（「3〜4人」）。

判定優先順位：**マンション種別・社名キーワード → 「○○の現場」 → 最初の語**。

**変更内容（タスクカードUI）：** 現場名と作業内容の `fontSize` を入れ替え。現場名 12→**15px**（主役）、作業内容 15→**12px**。

**注意：** 「最初の語＝現場名」のため、作業内容から話し始めると現場名に誤分類される。推奨の発話順は **現場名 → 日付 → 作業内容 → 人数**。違ったらプレビュー/編集で修正可。

## 重要な知識・注意点

### 音声一括登録はローカル解析（APIキー・課金なし）

- 解析はブラウザ内の `parseVoiceLocal` で完結。Anthropic等の外部APIは使わない（ユーザー方針：課金しない）。
- **この機能をAPI/AI方式で復活させないこと。** 精度を上げたい場合は `DATE_RE` や各抽出正規表現（人数・現場名・作業内容）を拡張する。
- 音声認識（文字起こし）自体は Web Speech API なので**ネット接続が必要**。完全オフラインで動くのは「解析」部分のみ。
- 相対日付の基準は実行時の `new Date()`（今日）。

### Web Speech API はモバイルで `continuous=true` を使わない

- スマホ（Android/iOSのChrome・Safari）は `continuous=true` で確定語を再報告し、語が大量に重複する。
- 連続的に聞きたい場合は **`continuous=false`（1発話ごと）＋ `onend` で自動再開** を使う（`startAiRec` 参照）。停止フラグはrefで持ち、確定テキストもrefに蓄積する。
- 単発入力（`MicButton`）は `continuous=false` / `interimResults=false` で重複しない。

### Service Worker のキャッシュ（フェーズ8で対策済み）

- `sw.js` は **HTMLはネットワーク優先＋オフライン時のみキャッシュ**。さらに `deploy.bat` がデプロイ毎にキャッシュ名を更新するため、通常はキャッシュ削除なしで最新が反映される。
- この方式を **初適用する時だけ** 旧SWが残っているスマホで一度キャッシュ削除/再読込が要る場合あり。
- `sw.js` を cache-first に戻さないこと（更新が出なくなる）。

### データ構造（タスクオブジェクト）

```js

  id: Number,
  siteName: String,        // 現場名
  detail: String,          // 作業内容・メモ（カードの見出しに使用）
  workers: String,         // 人数（例: "2〜3人"）
  status: "未着手" | "進行中" | "完了",
  dateMode: "期間指定" | "候補日",
  dateFrom: String,        // YYYY-MM-DD
  dateTo: String,          // YYYY-MM-DD（期間指定のみ）
  candidateDates: String[], // YYYY-MM-DD[]（候補日のみ）
  createdAt: Number,       // Date.now()
}
```

1. `sw.js` のキャッシュ名を現在時刻に自動更新（スマホのキャッシュ対策）
2. `git add -A` → `git commit` → `git push origin main`
3. 1〜2分で `https://ueorx06-cyber.github.io/genba/` に反映

**リポジトリ：** `github.com/ueorx06-cyber/genba`（`main`ブランチからPages配信）。
**認証：** 初回プッシュで Git Credential Manager にログイン済み。以降は不要。
**公開対象：** `index.html / manifest.json / sw.js / icon-192.png / icon-512.png / CLAUDE.md`。
`.claude/`（ローカル設定）・`deploy.bat`・スクショは `.gitignore` で除外。

> 手動でやる場合：`git add -A; git commit -m "..."; git push origin main`
> Claudeに「デプロイして」と頼めば代わりにpushも可能。
